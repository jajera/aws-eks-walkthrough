## Overview

The <Tooltip term="eks" /> control plane is managed by AWS — you are responsible only for the
worker nodes. Nodes run in private subnets created in the Networking batch and communicate
with the control plane over the VPC.

## Repository layout

<FileTree>

## Steps

<Steps>

1. **Create IAM roles.** <Badge text="Required" variant="tip" size="small" />

   The cluster needs two <Tooltip term="iam" /> roles: one for the control plane and one for
   the node group.

   ```hcl title="modules/eks/main.tf"
   resource "aws_iam_role" "cluster" {
     name               = "${var.name}-cluster"
     assume_role_policy = data.aws_iam_policy_document.cluster_assume.json
   }

   resource "aws_iam_role_policy_attachment" "cluster" {
     role       = aws_iam_role.cluster.name
     policy_arn = "arn:aws:iam::aws:policy/AmazonEKSClusterPolicy"
   }
   ```

2. **Provision the EKS cluster.** <Badge text="Required" variant="tip" size="small" />

   ```hcl title="modules/eks/main.tf"
   resource "aws_eks_cluster" "main" {
     name     = var.name
     role_arn = aws_iam_role.cluster.arn
     version  = var.kubernetes_version

     vpc_config {
       subnet_ids              = var.private_subnet_ids
       endpoint_private_access = true
       endpoint_public_access  = true
     }
   }
   ```

   <Aside type="tip">
     Set `endpoint_public_access = false` after initial setup if you want to restrict API server
     access to within the VPC only.
   </Aside>

3. **Create a managed node group.** <Badge text="Required" variant="tip" size="small" />

   ```hcl title="modules/eks/main.tf"
   resource "aws_eks_node_group" "main" {
     cluster_name    = aws_eks_cluster.main.name
     node_group_name = "${var.name}-nodes"
     node_role_arn   = aws_iam_role.nodes.arn
     subnet_ids      = var.private_subnet_ids

     scaling_config {
       desired_size = var.node_desired
       max_size     = var.node_max
       min_size     = var.node_min
     }

     instance_types = [var.node_instance_type]
   }
   ```

4. **Apply and update kubeconfig.** <Badge text="Required" variant="tip" size="small" />

   ```bash frame="terminal" title="walkthrough/"
   terraform apply
   ```

   Then update your local kubeconfig:

   <Tabs syncKey="tool">
     <TabItem label="AWS CLI">
       ```bash frame="terminal"
       aws eks update-kubeconfig \
         --name <cluster-name> \
         --region <region>
       ```
     </TabItem>
     <TabItem label="Terraform output">
       ```bash frame="terminal"
       terraform output kubeconfig_command
       # copy and run the printed aws eks update-kubeconfig command
       ```
     </TabItem>
   </Tabs>

5. **Verify cluster connectivity.** <Badge text="Required" variant="tip" size="small" />

   ```bash frame="terminal"
   kubectl get nodes
   kubectl get pods -A
   ```

   All nodes should show `Ready` and system pods should be `Running`.

</Steps>
